# Development

How to build, run, test, and contribute to this project locally. The audience is a new contributor (human or agent) who needs to go from clone to green build with minimal friction.

## What belongs here

- **Getting-Started.md** — first-run: prerequisites, install, run, test, verify.
- **Environment.md** — dev environment setup: language versions, env vars, local services.
- **Testing.md** — how to run each kind of test, where tests live, how to add new ones.
- **Conventions.md** — code style, commit style, PR process, naming.
- **Troubleshooting.md** — common setup issues and their fixes.
- **Completed-Features.md** — reverse-chronological log of shipped features, patches, and infra changes (maintained by `documenter` at `/release`).

## What does not belong here

- **Production operations** → [`operational/`](../operational/README.md).
- **Product intent and user-facing rationale** → [`design/`](../design/README.md).
- **System structure and architectural decisions** → [`architecture/`](../architecture/README.md).
